##### 课后思考

分析一下下面这个 add() 函数的时间复杂度。

```java
// 全局变量，大小为 10 的数组 array，长度 len，下标 i。
int array[] = new int[10]; 
int len = 10;
int i = 0;

// 往数组中添加一个元素
void add(int element) {
   if (i >= len) { // 数组空间不够了
     // 重新申请一个 2 倍大小的数组空间
     int new_array[] = new int[len*2];
     // 把原来 array 数组中的数据依次 copy 到 new_array
     for (int j = 0; j < len; ++j) {
       new_array[j] = array[j];
     }
     // new_array 复制给 array，array 现在大小就是 2 倍 len 了
     array = new_array;
     len = 2 * len;
   }
   // 将 element 放到下标为 i 的位置，下标 i 加一
   array[i] = element;
   ++i;
}
```

最好情况下，数组有空闲，直接添加，时间复杂度 O(1)

最坏情况下，数组满，需要重新申请空间，赋值原来数组元素后再添加，假设数组大小为 n ，时间复杂度为 O(n)

均摊时间复杂度，一次耗时操作 O(n)，后面有 n 次不耗时的 O(1)，均摊时间复杂度 O(1)
