#### 03 | 复杂度分析（上）：如何分析、统计算法的执行效率和资源消耗？

数据结构和算法就是要解决“快”和“省”的问题，所以复杂度分析是必要的。

##### 事后统计法

把代码跑一边，通过统计和监控得到算法的执行时间和占用内存大小。

这种方法的测试结果有一定局限性：

依赖测试环境，CPU性能等

受数据规模影响大，

> 这种方法也是判断算法性能很有效的措施，比如一些性能基线测试。一些刷题网站评判结果时也是用这样的方法。

##### 渐进时间/空间复杂度分析

代码执行时间/空间随着数据规模增长的变化趋势，一种粗略但简单有效的估计。

##### 时间复杂度分析

时间复杂度分析中的一些假设：

忽略不同的运算在时间上的差异，统一认为是单位时间，比如加法、乘法、比较等

分析方法：

- 只关注循环执行次数最多的一段代码
- 加法法则：总复杂度等于量级最大的那段代码的复杂度
- 乘法法则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积

几种常见的时间复杂度

多项式量级：

```
O(1) 常量阶
O(logn) 对数阶
O(n) 线性阶
O(nlogn) 线性对数阶
O(n^2) 平方阶
O(n^3) 立方阶
O(n^k) k次方阶
```

非多项式量级：

```
O(2^n) 指数阶
O(n!) 阶乘阶
```

> 随着数据规模 n 的增大，非多项式量级的时间复杂度会急剧增加，这是一种比较低效的算法
